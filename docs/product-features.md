# 产品功能说明

最后更新：2026-06-14

## 核心用户流程

项目围绕小说创作者的日常工作流展开：

1. 在资料库中维护人物、势力、事件、线索和世界规则。
2. 在大纲中选择当前章节，明确章节目标、冲突和禁用内容。
3. 在 AI 页面发起写作或提问。
4. 系统自动为 AI 组织有限资料上下文。
5. AI 返回章节草稿、解释、界面动作或资料变更建议。
6. 用户检查待审批变更，批准后写入资料库。

这个流程强调“创作者最终确认”，AI 不直接拥有资料库写权限。

## AI 页面

AI 页面是创作闭环的主入口，形态接近聊天式写作工作台。

已经落地的能力：

- 展示当前章节目标、引用资料、关系提示和时间线提示。
- 在 DeepSeek 配置完成后生成章节草稿。
- 支持用户自然语言询问资料、打开章节或打开资料弹窗。
- 展示 `AIContextBundle`，让用户知道本次 AI 使用了哪些资料。
- 展示 `AIProposedChange` 待审批列表。
- 支持批准或拒绝 AI 资料变更。

安全边界：

- 未配置 DeepSeek 时，不生成伪造演示草稿。
- AI 请求只发送资料索引和少量命中详情。
- 资料修改必须进入审批列表。

## 资料库页面

资料库页面负责沉淀长期设定。

资料分类包括：

- 人物
- 家族/势力
- 地点/地理
- 事件
- 物品/线索
- 世界观/规则
- 卷章/剧情
- 历史/年表

已经落地的能力：

- 按分类筛选资料。
- 点击资料打开详情弹窗。
- 使用右上角快捷按钮新建资料。
- 从详情弹窗进入完整编辑。
- 根据资料类型展示不同表单字段。
- 支持重要度、状态、标签、时间线、引用章节等通用字段。
- 支持人物和家族/势力之间的结构化关系维护。

## 大纲页面

大纲页面用于管理章节级写作计划。

已经落地的能力：

- 展示第 2 卷示例章节。
- 维护章节标题、目标、冲突、禁用内容和状态。
- 点击章节查看详情。
- 新建章节计划。
- 从章节计划发起 AI 写作上下文。

章节数据会参与 AI 上下文构建，尤其是 `goal`、`conflict` 和 `taboo` 字段。

## 关系/时间线页面

关系页面用于查看资料之间的连接，并辅助发现设定冲突。

已经落地的能力：

- 展示人物、势力、事件、物品和规则之间的关系节点。
- 打开某个节点的关系详情。
- 编辑关系资料。
- 展示关键时间点和引用章节。

关系类型包括：

- `member`：成员
- `ally`：盟友
- `enemy`：敌对
- `related`：关联
- `owner`：持有
- `affects`：影响
- `belongsTo`：归属

## 设置页面

设置页面管理运行环境和个性化配置。

已经落地的能力：

- DeepSeek API Key、Base URL 和模型配置。
- API 连接测试。
- 三套主题切换。
- 本地 JSON 导出。
- 样例数据重置。
- 本地服务模式下自动读取服务器配置状态。

内置主题：

- 清爽知识库：默认浅色，适合资料维护。
- 夜间创作台：深色沉浸，适合长时间写作。
- 专业资料中枢：高密度资料、关系和审批场景。

## 当前产品状态

项目已经具备可演示、可测试的 V1 原型能力。它适合继续用于产品评审、交互验证、Figma 回填和后续正式工程化拆分。

暂未覆盖的方向包括账号体系、云端同步、多作品管理、多人协作、正式权限系统和生产级数据存储。
