# 项目总览

最后更新：2026-06-14

## 项目定位

`笔维斯手机端原型` 是一个本地可运行、可点击的高保真移动端原型。它服务于长篇小说创作者，尤其是需要持续维护人物、势力、地点、事件、物品、世界观规则和章节大纲的创作场景。

项目当前的重点不是做完整商业化产品，而是验证 V1 核心闭环：

1. 创作者维护本地资料库。
2. AI 只读取与当前章节相关的有限上下文。
3. AI 生成章节草稿或回答创作问题。
4. AI 对资料库提出新增、修改、关联或删除建议。
5. 用户审核批准后，变更才写入资料库。

## 解决的问题

长篇小说写作常见难点是资料越来越多，但写作时很难稳定引用正确设定。AI 如果直接接触整库内容，既容易上下文过载，也容易把未经确认的灵感直接混入正式设定。

本项目用“资料库 + 有界 AI 上下文 + 人工审批”的方式解决三个问题：

- 资料统一沉淀：角色、势力、事件、线索、规则和章节计划有固定入口。
- AI 引用可追踪：AI 写作前会构建 `AIContextBundle`，界面展示本次引用的资料。
- 设定变更可控：AI 只能提交 `AIProposedChange`，不能绕过用户直接写入资料库。

## 当前形态

项目是一个原生 HTML/CSS/JavaScript 单页应用，外层模拟手机界面，右侧提供桌面端说明。它可以直接打开 `index.html` 作为纯前端原型，也可以通过 `npm run dev` 启动本地 Node 服务，获得本地 JSON 资料接口和 DeepSeek 代理能力。

当前已有主要页面：

- `AI`：ChatGPT 式创作区、上下文展示、章节草稿、待审批变更。
- `资料库`：分类浏览、资料详情、新建资料、编辑资料、结构化关系维护。
- `大纲`：章节目标、冲突、伏笔和禁用内容维护。
- `关系`：资料节点关系、关键时间线和关系编辑入口。
- `设置`：主题、DeepSeek API、本地数据导出和样例数据重置。

## 示例故事资料

样例内容使用长篇奇幻故事《云上王朝》：

- 主角：沈青岚，灵契师，调查云氏家族密约。
- 核心势力：云氏家族，王都旧贵族，保存旧王时代灵契档案。
- 核心事件：星陨宴，暗杀失败并暴露多方秘密联系。
- 核心线索：旧王印，可激活古老灵契。
- 世界规则：灵契记录真实誓约残响，能读取但不能伪造。

这些样例不是单纯占位数据，而是用于验证资料分类、关系推导、章节引用和 AI 审批流程是否能跑通。

## 项目价值

对产品验证来说，它能快速证明“AI 写作助手不是直接替作者改设定，而是作为可审计的资料协作者”这一方向是否成立。

对设计交付来说，它在 Figma 写入额度恢复前提供了本地可操作版本，方便继续打磨信息架构、移动端交互和主题视觉。

对开发实现来说，它已经具备可迁移的数据契约、API 雏形、测试用例和 DeepSeek 接入边界，后续可以逐步替换为正式前后端架构。
